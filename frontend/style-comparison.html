Light — LoginPage, ProposalsPage

These serve different purposes: dark = terminal/shell commands, light = inline metadata/IDs. Both are fine to keep as two variants: .code-block-dark and .code-inline.

Summary: My Recommendations

Element Recommendation Rationale
Primary button A (8px 16px, 13px, r6) Largest target, most readable
Secondary button A (warm + border) Matches palette, border gives affordance
Small/inline button D (white + outline, 11px) Good for table rows, compact UI
Danger button B (theme token) Single source of truth for red
Error text D (#c4493a = status-red) Already a theme token
Input fields E (proposed standard) Warm border token, consistent r6
Dividers footer-border / sidebar-border Two tiers: subtle + strong, both warm
Code blocks Keep both dark + light Different semantic purposes
Status pills B (theme tokens) Warmer, single source of truth
